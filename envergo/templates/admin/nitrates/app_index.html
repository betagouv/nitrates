Rôle de chaque modèle

🌳 ArbreDeDecision
Source de vérité runtime de la moulinette. JSON YAML versionné (draft / active / archive). Édité via le viewer /admin/nitrates/arbre-decision/.
🌱 GroupeCultureUI
Regroupement UX uniquement pour le 1er select de la cascade formulaire (« Culture d'hiver », « Prairies ou luzerne »…). Aucune logique métier ne branche dessus.
🌾 Culture
Culture finale choisie par l'utilisateur (« Colza », « Maïs »…). Sert de mapper entre la cascade UI et la branche d'arbre empruntée (FK branche_culturale + occupation_sol).
🌿 BrancheCulturale
Identifiant de sous-arbre du YAML (« colza », « culture_printemps »…). Plusieurs cultures peuvent emprunter la même branche. C'est ce sur quoi l'arbre branche réellement au niveau sous_culture.
💩 Fertilisant
Fertilisant utilisateur (« Fientes de volailles », « Lisier porcin »…). Catégorie + type réglementaire (type_0/Ia/Ib/II/III) en choices figés par l'arrêté. La liste est dynamique.
📜 CodePrescription
Prescription affichée en résultat de simulation (PC1, PC2…). Lié optionnellement à une NoteReglementaire.
📑 NoteReglementaire
Note de l'arrêté PAN (note_5, note_7…). Porte la liste des régions / départements concernés (utilisé pour résoudre les zones type « note_5 »).
🌸 EvenementPhenologique
Borne flottante de période (« brunissement_des_soies », « derniere_coupe_luzerne »…), avec date conventionnelle JJ/MM pour le rendu calendrier.
